Typescript Modules

Modules in Javascript

We have been using Javascripts ES Modules format to handle modules using the import/export syntax

  • In TypeScript any file containing a top-level import or export is considered a module
  • You can make TypeScript treat a file as a module by adding export {{ '{' }}}; to it
  • TypeScript adds the ability to import and export types on top of ES Modules functionality
        
// @filename: animal.ts
export type Cat = {{ '{' }} breed: string; yearOfBirth: number };
export type Dog = {{ '{' }} breeds: string[]; yearOfBirth: number };
export type Bird = {{ '{' }} breed: string; yearOfBirth: number; color: string };

// @filename: main.ts
import type {{ '{' }} Cat, Dog } from "./animal.js";    //mass import with import type
import {{ '{' }} type Bird } from "./animal.js";        //inline type import

Using ES6 modules in the browser

The default resolution strategy set in the tsconfig is Node - which uses CommonJs for module resolution.

To use the ES6 syntax of Import / Export with Typescript, first define your module type to es6 in the tsconfig

Also make sure to import your main js file in the index.html with type="module" set

    
// tsconfig.json
{{ '{' }}
    "compilerOptions": {{ '{' }}                             
        "module": "es6"  // Tells typsecript to use the classic resolution strategy                                   
    }
}   
    
    
<!-- index.html -->
<script src="main.js" type="module"></script>
    
    
// main.ts
import Example from './example.js'

Note that the import file example.js references the javascript version, not the typescript

By importing Example without brackets, we are assuming it is the default import. Brackets are necessary for any exports that are not default
